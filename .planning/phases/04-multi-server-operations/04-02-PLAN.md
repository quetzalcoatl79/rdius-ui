```

```

}

  <action>
    **1. Server API client** (`frontend/src/lib/server-api.ts`):
    Follow the exact pattern from `radius-api.ts` and `logs-api.ts` — use `apiFetch` from `@/lib/api`.

    Functions:
    - `getServers(): Promise<ServerResponse[]>` — GET /servers
    - `getServer(id: string): Promise<ServerResponse>` — GET /servers/{id}
    - `createServer(data: ServerCreate): Promise<ServerResponse>` — POST /servers
    - `updateServer(id: string, data: ServerUpdate): Promise<ServerResponse>` — PUT /servers/{id}
    - `deleteServer(id: string): Promise<void>` — DELETE /servers/{id}
    - `restartServer(id: string): Promise<{success: boolean, message: string}>` — POST /servers/{id}/restart
    - `getServerStatus(id: string): Promise<ServerStatus>` — GET /servers/{id}/status
    - `getServerHealth(id: string): Promise<ServerHealth>` — GET /servers/{id}/health

    Export TypeScript interfaces at the top: ServerResponse, ServerCreate, ServerUpdate, ServerStatus, ServerHealth (matching the backend schemas listed in interfaces section above).

    **2. Audit API client** (`frontend/src/lib/audit-api.ts`):
    - `getAuditLogs(params: AuditLogFilter): Promise<PaginatedResponse<AuditLogEntry>>` — GET /audit with query params built from the filter object. Omit null/undefined params from the query string.
    - Export TypeScript interfaces: AuditLogEntry, AuditLogFilter (user_email, action, resource_type, date_from, date_to, page, page_size).
    - Use the PaginatedResponse type (items: T[], total: number, page: number, page_size: number) — match the pattern from existing API clients.

    **3. Server context** (`frontend/src/lib/server-context.tsx`):
    - 'use client' directive at top.
    - Create React context with `createContext`.
    - `ServerProvider` component wraps children:
      - State: `selectedServerId: string | null`, `servers: ServerResponse[]`, `loading: boolean`
      - On mount: fetch servers list via `getServers()`, set state. If servers exist and no selection, auto-select the first one.
      - Persist selectedServerId to `localStorage` key `"radius-ui-server-id"`. On mount, restore from localStorage if the server still exists in the list.
      - Expose: `selectedServerId`, `selectedServer` (full object), `servers`, `setSelectedServerId`, `loading`, `refreshServers()`.
    - `useServer()` hook: `useContext(ServerContext)` with error if used outside provider.
    - This context will later be used by existing pages to scope API calls (SRV-03 — future enhancement, not in this plan's scope since it requires modifying all existing API calls).

    **4. Server selector component** (`frontend/src/components/layout/server-selector.tsx`):
    - 'use client' directive.
    - Uses `useServer()` hook.
    - Renders a native `<select>` dropdown (same pattern as Phase 03-02 decision: native HTML select for simple filters — avoids base-ui Select complexity).
    - Shows server name for each option. Shows a "Chargement..." disabled option while loading.
    - If no servers registered, show "Aucun serveur" disabled option.
    - On change: call `setSelectedServerId(value)`.
    - Styled with Tailwind: `w-full px-3 py-2 rounded-md bg-muted text-sm border border-border`.
    - Add a small label above: "Serveur actif" in text-xs text-muted-foreground.

    **5. Update sidebar** (`frontend/src/components/layout/sidebar.tsx`):
    - Import and render `<ServerSelector />` in the sidebar, positioned between the logo section and the navigation items.
    - Add a horizontal divider (`border-b border-border`) below the selector.
    - Add two new nav items:
      - `{ href: '/servers', label: 'Serveurs', icon: Server }` — under a new "Administration" section label
      - `{ href: '/audit', label: "Journal d'audit", icon: ScrollText }` — in the same Administration section
    - Import `ScrollText` from `lucide-react`.
    - Wrap the app layout with `<ServerProvider>` — if the layout already exists at `frontend/src/app/(dashboard)/layout.tsx`, add `<ServerProvider>` inside the existing auth provider. If it is in the sidebar itself, wrap at the appropriate level so both sidebar and page content have access.
