    **1. DockerService** (`backend/app/services/docker_service.py`):
    Create a static-method service class (same pattern as NasService) that communicates with Docker via httpx through the socket proxy. Import `settings` from `app.core.config`.

    Methods:
    - `async get_container_by_label(docker_container_id: str) -> dict | None`:
      - Build filter: `{"label": ["radius-ui.instance={docker_container_id}"]}`
      - GET `{settings.docker_socket_url}/containers/json?filters={filters}`
      - Return first container dict or None
      - Timeout: 5 seconds

    - `async restart_container(docker_container_id: str) -> bool`:
      - Find container via get_container_by_label
      - POST `{settings.docker_socket_url}/containers/{container_id}/restart?t=10`
      - CP-5 mitigation: always use restart (not HUP). The t=10 gives FreeRADIUS 10s for graceful shutdown
      - Return True on 204/200, False on failure
      - Log with logger.info on success, logger.error on failure

    - `async get_container_status(docker_container_id: str) -> dict`:
      - Find container via get_container_by_label
      - If not found: return {"status": "not_found"}
      - GET `{settings.docker_socket_url}/containers/{container_id}/json`
      - Extract from response: State.Status, State.StartedAt, State.FinishedAt
      - Calculate uptime_seconds from StartedAt to now (if running)
      - Return dict matching ServerStatus schema fields

    - `async get_container_health(docker_container_id: str) -> dict`:
      - Find container via get_container_by_label
      - If not found: raise HTTPException 404
      - GET `{settings.docker_socket_url}/containers/{container_id}/stats?stream=false`
      - PT-3 mitigation: the `stream=false` ensures a single snapshot, not a stream
      - Parse Docker stats response to extract:
        - CPU%: `(cpu_delta / system_delta) * num_cpus * 100`
        - Memory: usage_bytes, limit_bytes, percentage
      - Return dict matching ServerHealth schema fields

    All methods wrap httpx calls in try/except httpx.HTTPError, logging errors and raising HTTPException 503 with clear messages.

    **1. Server endpoints** (`backend/app/api/v1/servers.py`):
    Create APIRouter. All endpoints require admin or super_admin role (use `require_role("admin", "super_admin")` from deps.py).

    Endpoints (per SRV-01, SRV-04, SRV-05, SRV-06):
    - `POST /` — Create server registration.
      - Body: ServerCreate. Returns ServerResponse (201).
      - Call AuditService.log(action="create", resource_type="server", resource_id=str(server.id), details={"name": data.name}).
      - Extract IP from request.client.host.
    - `GET /` — List all servers. Returns list[ServerResponse]. Viewer+ access (require_role("viewer", "operator", "admin", "super_admin")).
    - `GET /{server_id}` — Get single server. Returns ServerResponse. Viewer+ access.
    - `PUT /{server_id}` — Update server. Body: ServerUpdate. Returns ServerResponse.
      - Audit log: action="update", resource_type="server".
    - `DELETE /{server_id}` — Delete server. Returns 204.
      - Audit log: action="delete", resource_type="server".
    - `POST /{server_id}/restart` — Restart FreeRADIUS container (SRV-04).
      - Look up server from DB to get docker_container_id.
      - Call DockerService.restart_container(server.docker_container_id).
      - Audit log: action="restart", resource_type="server" (per AUDIT-04).
      - Return {"success": true, "message": "Container restart triggered"} or 503 on failure.
    - `GET /{server_id}/status` — Server status (SRV-05).
      - Call DockerService.get_container_status(server.docker_container_id).
      - Return ServerStatus. Viewer+ access.
    - `GET /{server_id}/health` — Server health metrics (SRV-06).
      - Call DockerService.get_container_health(server.docker_container_id).
      - Return ServerHealth. Viewer+ access.

    For every endpoint that takes server_id, look up the server first and return 404 if not found.

    **2. Audit endpoints** (`backend/app/api/v1/audit.py`):
    Create APIRouter. All endpoints require admin or super_admin role (per AUDIT-03).

    Endpoints:
    - `GET /` — List audit logs with optional query params: user_email, action, resource_type, date_from (ISO datetime), date_to (ISO datetime), page (int, default 1), page_size (int, default 50).
      - Call ServerService.list_audit_logs() with filters.
      - Return PaginatedResponse[AuditLogResponse] (reuse existing PaginatedResponse generic from radius schemas, or create equivalent).

    **3. Wire into router** (`backend/app/api/v1/router.py`):
    - Import servers router and audit router.
    - `api_router.include_router(servers_router, prefix="/servers", tags=["servers"])`
    - `api_router.include_router(audit_router, prefix="/audit", tags=["audit"])`

    **Important implementation details:**
    - Every mutating endpoint (POST, PUT, DELETE on any resource) should call AuditService.log() AFTER the successful DB operation but BEFORE committing. This ensures the audit log and the data change are in the same transaction.
    - Use `request: Request` parameter in endpoint signatures to extract `request.client.host` for ip_address.
    - The PaginatedResponse generic already exists in `backend/app/schemas/radius.py`. Import and reuse it for audit logs.
